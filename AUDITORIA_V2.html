1. Resumen Ejecutivo

El proyecto Nomada es un frontend desarrollado en React 19 + Vite 8 + Tailwind CSS para una agencia de viajes. Sigue el patrón Atomic Design con componentes organizados en atoms → molecules → organisms → pages. Se conecta a un backend REST en http://localhost:8080 e incluye un servidor Express mockeado integrado.

4. Problemas Críticos

🔴 CRIT-1: Typo en URL de login

Archivosrc/services/authService.js:20
ProblemaUsa /api/authentication/login (typo "authentication")
Realidadserver.js expone /api/auth/login
ImpactoEl login no funciona contra el servidor mock. Petición 404.
SoluciónCambiar authentication → auth

🔴 CRIT-2: Inconsistencia de cliente HTTP

Archivossrc/services/api.js vs todos los servicios
ProblemaapiClient con axios configurado con interceptores pero ningún servicio lo usa
RealidadTodos los servicios usan fetch nativo con headers manuales
ImpactoInterceptor de renovación de token no se ejecuta. Código duplicado.
SoluciónUnificar todos los servicios para usar apiClient o crear helper centralizado

🔴 CRIT-3: 9 componentes CRUD obsoletos

ArchivosBookingsCRUD.jsx, BusesCRUD.jsx, DriversCRUD.jsx, EmployeesCRUD.jsx, HotelsCRUD.jsx, OffersCRUD.jsx, TravelsCRUD.jsx, TripSegmentsCRUD.jsx, UsersCRUD.jsx
ProblemaReemplazados por EntityTable genérico pero aún presentes
Impacto~20KB de código muerto en el bundle. Algunos aún importados.
SoluciónEliminar los 9 archivos y limpiar imports

🔴 CRIT-4: Credenciales Cloudinary en .env

Archivo.env
ProblemaContiene CLOUDINARY_API_KEY y CLOUDINARY_API_SECRET reales
RiesgoAlto si alguien comitea el .env accidentalmente
SoluciónRotar las credenciales y nunca incluirlas en el repo

5. Problemas Importantes

🟡 IMP-1: Bug de scroll en Modal

El modal no permite scroll cuando el contenido excede la ventana. Formularios largos dentro de modales no son accesibles. Fix documentado en docs/fix-modal-scroll.md sin aplicar.

🟡 IMP-2: Checkout frágil — depende de location.state

CheckoutPage.jsx:57 recibe datos del viaje exclusivamente por location.state. Si el usuario refresca la página, pierde todo el contexto y es redirigido.

🟡 IMP-3: Validaciones de negocio faltantes

ValidaciónUbicaciónImpacto
No reservar si autobús completoCheckoutPage.jsxOverbooking
No reservar si hotel completoCheckoutPage.jsxOverbooking
No vender viajes pasadosCheckoutPage.jsx + AdminViajes ya ocurridos
Email válido en registroAuthPage.jsxEmails inválidos

🟡 IMP-4: .env.example desactualizado

Solo incluye VITE_API_URL. Faltan: CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET, JWT_SECRET.

🟡 IMP-5: server.js — Contraseñas hardcodeadas

6 usuarios con contraseñas en texto plano: admin123, user123, 1234 (x4). Riesgo de seguridad en el código mock.

13. Recomendaciones Prioritarias

🔴 Críticas (Antes de la entrega)

1. Corregir typo en authService.js

Cambiar /api/authentication/login → /api/auth/login en la línea 20.

2. Eliminar componentes CRUD obsoletos

Borrar los 9 archivos CRUD individuales y limpiar imports en páginas/rutas.

3. Arreglar bug de scroll en Modal

Aplicar fix documentado en docs/fix-modal-scroll.md (overflow-y + max-height).

4. Agregar validaciones de negocio en checkout

Capacidad autobús, fechas pasadas, disponibilidad hotel antes de confirmar reserva.

🟡 Importantes

5. Unificar cliente HTTP

Migrar servicios a apiClient de axios o crear helper centralizado con fetch.

6. Persistir estado de checkout

Usar URL params + refetch en lugar de location.state.

7. Actualizar .env.example

Incluir CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET, JWT_SECRET.

8. Ampliar tests

Tests de páginas críticas, CRUDs, flujo E2E completo (login → buscar → checkout).

9. Limpiar archivos de merge conflict

Eliminar package-lock_*_784.json.

10. Actualizar README

Reflejar estructura actual del proyecto (services/, hooks/, utils/) y eliminar referencias obsoletas.

🟢 Opcionales

11. Eliminar dependencias no usadas

dompurify, considerar eliminar axios si no se va a usar.

12. Usar librería jwt-decode

Para decodificación JWT en lugar del método manual actual.

13. Agregar validación de email con regex

En AuthPage.jsx para evitar emails inválidos.

14. Implementar loading states en botones de submit

Feedback visual durante operaciones asíncronas.

15. Implementar debounce en buscadores

Optimizar rendimiento de búsquedas en tiempo real.

16. Agregar campo de experiencia en DriverForm

Consistencia con datos mock existentes.

17. Conectar PersonalAreaPage con datos reales

Actualmente solo UI sin conexión a servicios.

18. Rotar credenciales Cloudinary

Las credenciales actuales en .env están potencialmente comprometidas.

15. Conclusión

Puntos Fuertes

  • Arquitectura Atomic Design bien implementada y organizada
  • CRUD genérico (EntityTable) que maneja 9 entidades
  • Componentes UI reutilizables, consistentes y con tests
  • Autenticación JWT con rutas protegidas (públicas vs admin)
  • Diseño responsive con Tailwind CSS
  • Subida de imágenes a Cloudinary implementada
  • Manejo de errores global con ErrorBoundary
  • 14 páginas funcionales que cubren casi todos los requisitos

Puntos Débiles

  • Error crítico en login: typo en URL de autenticación
  • Duplicidad HTTP: axios configurado pero no usado
  • Código muerto: 9 componentes CRUD obsoletos
  • Validaciones de negocio incompletas
  • Checkout frágil: dependiente de location.state
  • Bug de scroll en Modal
  • Cobertura de tests insuficiente
  • Documentación desactualizada

Veredicto

El proyecto Nomada tiene una base técnica sólida con buena arquitectura y organización. La mayoría de los requisitos funcionales están implementados. Sin embargo, existen 4 problemas críticos que deben resolverse antes de la entrega final, siendo el más urgente el typo en la URL de login que impide la autenticación.

Con las correcciones prioritarias señaladas, el proyecto puede cumplir con todos los criterios de aceptación del briefing.
